# Generate Documentation Skill

Creates comprehensive project documentation for software projects.

## Documentation Types

- **README.md** — Project overview, features, installation, quick start,
  usage examples, configuration, contributing, license
- **API Documentation** — Endpoints, request/response formats, auth,
  error codes, examples
- **User Guide** — Step-by-step instructions, workflows, tips, FAQ,
  troubleshooting
- **Contributing Guide** — Dev setup, code style, PR process, testing,
  review process

## Process

1. **Analyze Project**: Examine code structure, dependencies, functionality
2. **Identify Audience**: Users, developers, or contributors
3. **Outline Content**: Create logical structure
4. **Write Sections**: Develop each section with appropriate detail
5. **Add Examples**: Include relevant code examples and use cases
6. **Review & Refine**: Ensure clarity, accuracy, completeness
7. **Format**: Proper Markdown with heading hierarchy, code blocks,
   tables, and links

## Output

Documentation in Markdown format, following these standards:
- Clear heading hierarchy (H1, H2, H3)
- Code blocks with syntax highlighting
- Tables for structured data
- Links to related sections
